# inkQuota 项目规划

## 1. 项目概述

一个 DIY 墨水屏桌面摆件，实时展示各大 AI 编程助手（Codex、Kimi、GLM 等）的额度信息，帮助用户合理安排 AI 编程时间。

---

## 2. 核心功能

### 2.1 显示内容
- **5小时窗口倒计时**：距离下一个额度重置的剩余时间
- **周额度剩余**：本周剩余可用额度（小时数）
- **总额度占比**：当前余额/总余额的百分比
- **账户余额**：实际金额（如 $12.3）
- **下次重置时间**：精确到秒的倒计时

### 2.2 状态指示
- RGB 指示灯根据额度紧张程度变色：
  - 🟢 绿色：充裕（>50%）
  - 🟡 黄色：一般（20%-50%）
  - 🟠 橙色：紧张（5%-20%）
  - 🔴 红色：告急（<5%）
  - 🟣 紫色快闪：正在刷新数据
  - 🔵 蓝色慢闪：配置/网络错误

### 2.3 支持的模型/服务

| 服务 | 额度类型 | 获取方式 | 优先级 |
|------|----------|----------|--------|
| **OpenAI Codex** | 5h窗口、周额度、余额 | Usage API | P0 |
| **Kimi (Moonshot)** | 5h窗口、余额 | 开放平台 API | P0 |
| **智谱 GLM** | 5h窗口、余额 | 手动配置 | P1 |
| **Claude** | 5h窗口、余额 | Console API | P1 |
| **Gemini** | 日额度、余额 | AI Studio API | P2 |
| **DeepSeek** | 余额、RPM限制 | 开放平台 API | P2 |
| **本地 Ollama** | GPU显存、队列 | 本地 HTTP | P3 |

### 3.1 最小系统（推荐入门）

```
├── WS2812B RGB灯珠 ×1

└── USB-C供电
```

### 3.2 进阶版本
```

├── 2.9寸 黑白/三色墨水屏
├── WS2812B RGB灯带 ×3（状态环）

├── 18650锂电池 + 充电模块
├── 3D打印外壳
└── 可选：温湿度传感器（显示环境信息）
```

---

## 4. 软件架构

### 4.1 两种部署模式

**模式A：Standalone（设备直连）**
```
ESP32 ←→ Wi-Fi ←→ 各平台API
```
- 优点：部署简单，无需额外设备
- 缺点：API Key存储在设备上

**模式B：Gateway（网关代理）**
```
ESP32 ←→ Wi-Fi ←→ 家庭网关(Raspberry Pi/软路由) ←→ 各平台API
```
- 优点：API Key安全，可统一处理认证
- 缺点：需要额外运行网关服务

### 4.2 代码结构
```
src/
├── main.py              # 主循环：联网→拉取→显示→休眠
├── config.py            # 配置文件读取
├── display.py           # 墨水屏驱动 + UI绘制
├── led.py               # WS2812 RGB控制
├── quota_fetcher.py     # 各平台额度拉取
├── utils.py             # 工具函数（时间、缓存、网络）
└── fonts/               # 字体文件
```

### 4.3 刷新策略
- 正常状态：每5-10分钟刷新一次
- 额度紧张时：每1分钟刷新
- 手动刷新：按钮触发
- Deep Sleep：刷新后进入休眠，RTC定时唤醒

## 7. 风险与注意事项

1. **API Key安全**：建议使用只读Key，或Gateway模式
2. **墨水屏寿命**：刷新次数有限（约10万次），避免频繁刷新
3. **Wi-Fi稳定性**：考虑断网重连和离线模式
4. **时区准确性**：需要NTP同步或手动设置
5. **功耗管理**：电池供电时需要精细的功耗控制
