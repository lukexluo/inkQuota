# inkQuota 项目计划

> 一个 DIY 墨水屏桌面摆件，实时展示各大 AI Coding Plan（阿里云百炼、智谱 GLM、Kimi、火山方舟等）的额度信息，帮助用户合理安排 AI 编程时间。

---

## 1. 项目概述

把各平台 AI 编程套餐的额度可视化，放在桌面一眼可见。用低功耗墨水屏保证 24h 常显，RGB 指示灯实时反馈额度健康度，像给 AI 编程习惯装了一个仪表盘。

---

## 2. 核心功能

### 2.1 显示内容

| 显示项 | 说明 | 数据来源 |
|--------|------|---------|
| **5h 窗口倒计时** | 精确到分钟的滑动窗口释放时间 | 本地滑动窗口计算 |
| **5h 剩余/占比** | 当前 5h 窗口剩余额度 + 百分比条 | 平台 API |
| **周剩余/占比** | 本周剩余额度 + 百分比条 | 平台 API |
| **月剩余/占比** | 本月剩余额度 + 百分比条 | 平台 API |
| **多模型聚合** | 同一屏幕展示 3-4 个模型卡片 | 多平台聚合 |
| **状态摘要** | 最紧张模型 + 建议操作 | 本地计算 |

> **占比设计**：每个模型卡片内嵌 **三层进度条**——外层 5h、中层周、内层月，用不同灰度或三色屏的红黑区分。

### 2.2 RGB 指示灯（可自定义）

支持 **1-3 颗 WS2812B** 或 **8 位灯环**，颜色映射完全可配置：

| 默认灯位 | 颜色 | 状态 | 触发条件 |
|---------|------|------|---------|
| **主灯（灯0）** | 🟢 绿色 | 充裕 | 5h 剩余 > 50% |
| **主灯（灯0）** | 🟡 黄色 | 警戒 | 5h 剩余 20%-50% |
| **主灯（灯0）** | 🔴 红色 | 紧张 | 5h 剩余 < 20% |
| **副灯（灯1）** | 🔵 蓝色 | 周额度告急 | 周剩余 < 30% |
| **副灯（灯1）** | 🟠 橙色 | 月额度告急 | 月剩余 < 30% |
| **呼吸灯（灯2）** | 🟣 紫色 | 数据刷新中 | 墨水屏更新时闪烁 |
| **全灯** | ⚪ 白色 | 离线/无网络 | WiFi 断开 |

> **自定义方式**：通过 `config.json` 修改颜色映射，支持 HSV 色值自定义。

## 4. 软件架构

### 4.1 两种部署模式

**模式A：Standalone（设备直连）**
```
ESP32 ←→ Wi-Fi ←→ 各平台 API
```
- 优点：部署简单，无需额外设备
- 缺点：API Key 存储在设备上

**模式B：Gateway（网关代理）**
```
ESP32 ←→ Wi-Fi ←→ 家庭网关（Raspberry Pi/软路由/NAS）←→ 各平台 API
```
- 优点：API Key 安全，可统一聚合多平台
- 缺点：需要额外运行网关服务

### 4.2 代码结构

```
src/
├── main.py              # 主循环：联网 → 拉取 → 绘制 → 休眠
├── config.py            # 配置读取（WiFi、API、颜色映射）
├── display.py           # 墨水屏驱动 + UI 渲染（Pillow → BMP）
├── led.py               # WS2812B RGB 控制（FastLED/MicroPython）
├── quota_fetcher.py     # 各平台额度拉取 + 滑动窗口计算
├── renderer.py          # 位图生成（进度条、卡片、倒计时）
├── fonts/               # 字体文件（Unifont / 中文点阵）
└── assets/              # 图标素材（Logo、箭头、状态图标）
```

### 4.3 刷新策略

| 状态 | 刷新频率 | 说明 |
|------|---------|------|
| 正常 | 每 5-10 分钟 | 墨水屏寿命保护 |

## 6. 关键算法：5h 滑动窗口

Coding Plan 的 5h 额度不是固定时间点重置，而是**滑动窗口**——每次请求在 5 小时后释放。

```python
from collections import deque
import time

class SlidingWindowQuota:
    """5h 滑动窗口额度计算器"""
    
    def __init__(self, quota_per_5h: int):
        self.quota = quota_per_5h
        self.requests = deque()  # (timestamp, count)
    
    def record_usage(self, count: int):
        self.requests.append((time.time(), count))
        self._expire_old()
    
    def _expire_old(self):
        cutoff = time.time() - 5 * 3600
        while self.requests and self.requests[0][0] < cutoff:
            self.requests.popleft()
    
    def get_remaining(self) -> int:
        self._expire_old()
        used = sum(c for _, c in self.requests)
        return self.quota - used
    
    def get_next_release(self) -> float | None:
        """返回下一个额度释放的时间戳"""
        if not self.requests:
            return None
        return self.requests[0][0] + 5 * 3600
    
    def get_countdown_str(self) -> str:
        """返回倒计时字符串如 '2h14m'"""
        nxt = self.get_next_release()
        if not nxt:
            return "--"
        diff = max(0, nxt - time.time())
        h, m = int(diff // 3600), int((diff % 3600) // 60)
        return f"{h}h{m:02d}m"
```

## 9. 风险与注意事项

1. **API Key 安全**：Coding Plan 的 Key 禁止 API 直接调用（仅限编程工具），建议用 Gateway 模式隐藏 Key
2. **墨水屏寿命**：刷新次数约 10 万次，避免频繁刷新（> 1次/分钟）
3. **Wi-Fi 稳定性**：断网后保留最后一次显示，LED 切白色提示
4. **时区准确性**：必须 NTP 同步，否则 5h 滑动窗口计算错误
5. **功耗管理**：电池供电时 Deep Sleep 功耗 < 10μA，屏幕断电保持画面
